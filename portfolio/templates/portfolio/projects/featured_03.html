Procedural Content Generator

This project procedurally generates a map in a text file, detects the walls, and records the wall coordinates in a file that is read by a script in Unity. Unity uses the coordinates to place cube objects, making the walls for the procedurally generated dungeon layout.

View code at : Github Repo

Project Details

Python is used to generate the map text file and record the coordinates. Unity is used to create the 3D objects with a C# script.


Step O1:

Creates the randomly generated area. This is done with randomly placed rectangles that overlap. Empty space is represented by '-' characters, and the interior area is denoted with 'x'.

Step O2:

This was the hard part. In order to delineate the walls, a moving-window algorithm was developed. This algorithm iterates through the whole map line-by-line until it detects a room area. Then it moves around the room area, changing the symbol from 'x' to '0'. A new file with just '0' characters for walls is created. While this file is being created, the algorithm records the (x,y) coordinates to a seperate data file for export to Unity.

Step O3:

A simple C# script is used to parse the data file into Unity data structures. A new cube object is created and placed at the corresponding coordinate in the 3D space.
